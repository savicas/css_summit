SELECTORS:

CSS3 es modular y la W3C es quien proporciona los grados de madurez que tiene Madurez W3Cy suelen seguir este proceso:

  • Nivel de Madurez en el proceso del trabajo: WD - working draft, es el borrador de trabajo que publica la W3C para que sea revisado por la comunidad, miembros de la W3C, público y demás org técnicas.
  • Nivel de Madurez del proceso de recomendación:
    1. CR - Candidate Recommendation, es un documento publicado que se ha revisado ampliamente x la W3C y que satisface los requerimientos técnicos.
    2. PR - Proposed Recommendation, es un informe técnico enviado al Comité Asesor del W3c más madurado, revisado por la W3C, aplicable y sólido técnicamente, para su aprobación final.
    3. REC - W3C Recommendation, es una especificación o un conjunto de directrices aprobadas por la W3C y de su director. Es la aprobación final del trabajo de recomendaciones anterior. NOTA: Las recomendaciones de la W3C son similares a las normas publicadas por otras org.
  • Nivel de Madurez en el proceso de finalización del informe técnico: Working Group Note es un informe publicado por la W3C para indicar que el trabajo ha terminado con un tema particular.
  • Nivel de Madurez en la edición de la recomendación: Proposed Edited Recommendation, es una recomendación publicada que ha sido revisada.
  • Nivel de madurez cuando se rescinde una recomendación: Rescinded Recommendation.
  • Nivel de Madurez para los informes de los grupos de interés y coordinación del grupo técnico: La W3C puede publicar notras de grupos de interés y de grupos de coordinación porque están estrechamete vinculados con la W3C Working Draft y se pueden complementar.

Selectores de Módulo

Recomendaciones propuestas por la W3C

En CSS3 los selectores tienen un sentido comun y es que buscan una mayor precisión sin necesidad de añadir clases. Estos son algunas pseudo-clases:

  • :firts-child --> con él puedes retocar el primer "hijo" de un parent element (por ejemplo, un grupo de párrafos. Ej de código: 
    div#wrapper p:first-child {font-size: 1.5em;}
    )
  • :last-child --> para retocar el último "hijo". PE: si quieres eliminar un borde del último párrafo para que no se vea. Ej de código: 
    ul#navigation li:last-child {border-bottom: none;}
  • :nth-child --> Nos sirve, por ejemplo, para escoger y remarcar distintas o todas las celdas de una tabla; Un Ej. de código para seleccionar todas las celdas impares: 
    tr:nth-child(odd) td {background-color:#ffff;}
    Un ejemplo en el que seleccionamos la celda 3: 
    tr:nth-child(3) td {background-color: #f0e9C5;}
    Más usos e información aquí
  • :nth-of-type --> Selecciona múltiples elementos de acuerdo a su posición en el árbol del documento pero sólo esos elementos que son iguales a la norma que se le aplica. Ej: queremos que todos los párrafos que justo antes tienen un "heading" tengan un fondo amarillo; El código sería así: 
     p:nth-of-type(odd) {background-color: yellow;}
  • :only-child --> encaja con el elemento si es el único elemento de su "parent". Código de ejemplo: 
    li {list-style-tipe: disc;} li: onli-child { list-style-tipe:none;} así, el li que sea distinto a los demás li no llevará el disc
  • :empty --> encaja con el elemento si este está vacío. Un ejemplo de código: 
     p {padding: 0 0 0 1em; margin: 0;} p:empty { padding: 0;}
    • Para elementos input:
  • :checked --> para un checkbox o radio button. Por ejemplo: 
    #agreeterms:checked { border: 2px solid blue;}
  • :enabled -->detecta el estado del elemento input. Código de ej: 
    input:enabled {color:#000;}
  • :disabled -->detecta el estado del elemento input. Código de ej: 
    input:disabled {color:#111;}
  • :not(selector) --> Es el pseudo-element de negación; Ejemplo de código: 
    p:not (.box) { padding:0 0 0 3em; margin: 0;} p.box {border: 1px solid blue; margin: 0 0 0 2em;}
  • :firt-letter --> Para modificar el primer carácter de la primera línea del texto. Ejemplo de código: 
    div#wrap:first-letter {color: red; font-size: 200%;}
  • :first line -->Formatea la primera línea del texto. Ej. de código: 
    div#wrap:first-line {color: red; font-size: 200%;}
  • :before -->Renderiza antes el contenido element que el resto de contenido. Ej. de código: 
    div id="content"> y en el CSS --> #content:before {content: "Start here:";}
  • :after --> Renderiza más tarde el contenido del element. Por ejemplo: 
    #content:after { content:"End here:";}
  • ::Selection -->contenido seleccionado por el usuario en el navegador. Código de ej: 
    div#wrapper p::selection {background-color: red;}
  • Combinators --> combinan selectores para algunos elementos
  • Child selector --> Selecciona todos los elementos que son hijos inmediatamente después de un padre específico. Por ejemplo:

  • Adjacent sibling (hermano adyacente) --> Selecciona elementos que son los hermanos adyacentes de un elemento. Por ejemplo: 
    div#wrapper h1+p {font-size: 1.5em;}
  • General Sibling (hermano gral) --> Selecciona los elementos que son hermanos del elemento. Código de ejemplo: 
    div#wrapper h2~p {color:red;}
    • Selectores de Atributos:
  • Seleccionan elementos basados en atributos. Por ejemplo: 
    form input[type="text"] {}      form input[type="submit"]{}
    Un ejemplo de codificación podría ser: 
     a[href ^="mailto:"] { padding-right: 20px; background-color:#555;}

Polyfills CSS:

	
Un Polyfill, o polyfiller, es un programa (o plugin) que proporciona la tecnología que esperas que el navegador proporcione de forma nativa. Esto es, que si el navegador no soporta alguna de las funcionalidades que le pedimos (gradientes, :first-child,...), el polyfill se ejecutará de manera que el navegador lo "soporte".
